# 📑 CRM Guide: Логика работы с лидами

Этот документ описывает внутреннюю логику CRM-модуля.

## 1. Лид -> Клиент
- **Лид**: Любая входящая заявка (квиз, форма, калькулятор). Каждой заявке присваивается уникальный ID сессии.
- **Клиент**: Создается автоматически или вручную при совпадении Телефона или Telegram ID. В карточке клиента отображается лента (Timeline) всех его лидов.

## 2. Канбан-система
- Каждая колонка — это статус в БД. 
- Перетаскивание карточки меняет статус лида.
- Новые лиды попадают в первую колонку и имеют визуальный индикатор (пульсирующую точку), пока не будут открыты в первый раз.

## 3. Заметки и Планирование
- Внутри карточки есть текстовое поле для свободных заметок.
- Поле "Следующий шаг" (Next Action) — краткий текст, который отображается прямо на карточке в канбане, чтобы сразу видеть план действий.
