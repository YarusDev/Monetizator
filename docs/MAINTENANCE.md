# 🔧 Инструкция по обслуживанию сайта

В этом руководстве описано, как самостоятельно изменять контент сайта без глубокого погружения в код.

## 📝 Изменение текстов и цен

### Секции и Блоки
Большинство текстов находится в папке `src/components/sections/`.
*   **Цены и Услуги**: Редактируйте `Services.tsx`. Ищите массив `services`.
*   **Кейсы**: Данные кейсов вынесены в `src/constants/cases.ts`. Там можно легко добавить новый кейс или изменить цифры прибыли.
*   **Вопросы квиза**: Редактируйте `Quiz.tsx`. Массив `questions` содержит вопросы, варианты ответов и "инсайты".

### Заголовки
Если нужно изменить заголовок секции, ищите компонент `SectionHeader` внутри файлов секций.

## 🖼 Работа с изображениями
Все картинки лежат в `public/assets/`.
*   Чтобы заменить фото, просто загрузите новый файл с тем же именем (например, `LogoCoin.png`) или обновите путь в коде компонента.
*   Рекомендуемый формат для фото: **WebP** или **PNG** (с прозрачностью для логотипов).

## 🧮 Калькулятор
Логика расчета прибыли находится в `src/components/sections/Calculator.tsx`. 
Если нужно изменить формулу (например, процент скрытой прибыли), ищите переменную `potentialProfit`.

## 🛠 Техническое обслуживание
1.  **Проверка типов**: Перед деплоем желательно запустить `npm run type-check`.
2.  **Обновление зависимостей**: Не обновляйте `lucide-react` или `motion` без проверки — это может сломать анимации.
3.  **Supabase**: Если данные перестали сохраняться, проверьте статус проекта в консоли Supabase и валидность ключей в `.env`.

---
*При возникновении критических ошибок — обращайтесь к технической документации в папке `Документация по сайту`.*
