# **Часть 1: Инфраструктура и DevOps (Antigravity Protocol)**

Этот раздел определяет технический фундамент системы и правила, по которым код попадает с твоего компьютера на сервер.

## **1.1. Регламент работы с Git и CI/CD**

Согласно протоколу «Антигравити», работа с кодом ведётся в изолированной среде с чётким разделением репозиториев.

* **Целевой репозиторий:** https://github.com/YarusDev/Monetizator.git.  
* **Метод деплоя (Subtree):** Категорически запрещено деплоить папку напрямую. Используется команда:  
  git subtree push \--prefix sites/monetizator-v2 monetizator main.  
* **Автоматизация (Workflow):** Файл deploy.yml для автоматического развёртывания должен находиться по пути: sites/monetizator-v2/.github/workflows/deploy.yml.

## **1.2. Технологический Стек (The Tech Core)**

Мы используем современные инструменты, которые обеспечивают мгновенную загрузку сайта и надёжную работу ИИ.

* **Frontend:** React \+ Vite (для максимальной скорости сборки).  
* **Стилизация:** Tailwind CSS (настройка под протокол «The Money Matrix»).  
* **Backend & Database:** Supabase (PostgreSQL). Поддерживает мульти-арендность через поле expert\_id в таблицах.  
* **ИИ-двигатель:** Vertex AI (Gemini 1.5 Pro) для анализа и генерации ответов.  
* **Голосовой ввод (Future):** Assembly AI (интеграция в слой обработки сообщений бота после реализации текстовой версии).

## **1.4. Стратегия Масштабируемости (Multi-tenancy)**

Несмотря на запуск под одного эксперта, база данных проектируется как платформа:

* **Expert ID:** Каждая запись в таблицах leads, products, content\_blocks и sources содержит идентификатор эксперта.  
* **Изоляция:** Правила RLS (Row Level Security) в Supabase настроены так, чтобы один эксперт никогда не увидел лидов другого.

# **Часть 3: Архитектура Данных (Supabase Core)**

Этот раздел описывает структуру базы данных PostgreSQL (внутри Supabase), логику серверных функций и правила безопасности.

## **3.1. Глобальная структура и Мульти-арендность (Multi-tenancy)**

Для реализации возможности масштабирования, в каждую таблицу вводится поле `expert_id`. Это позволяет хранить данные сотен экспертов в одной базе, обеспечивая их полную изоляцию через политики RLS (Row Level Security).

### **Основные таблицы БД:**

#### **1\. `experts` (Реестр экспертов)**

Хранит настройки личности и конфигурацию ИИ для каждого владельца системы.

* `id`: uuid (PK)  
* `name`: text (Имя эксперта)  
* `dna_prompt`: text (Системная инструкция для Vertex AI)  
* `settings`: jsonb (Цветовые настройки, ссылки на соцсети)

#### **2\. `m_content_blocks` (CMS сайта)**

Источник правды для фронтенда. Никакого хардкода — все тексты здесь.

* `id`: uuid (PK)  
* `expert_id`: uuid (FK)  
* `slug`: text (Уникальный код блока: `hero`, `cases`, `calc`)  
* `content`: jsonb (Заголовки, тексты, ссылки на изображения)  
* `order_index`: int (Порядок отображения блоков)  
* `is_active`: bool (Тумблер отображения блока)

#### **3\. `leads` (Центральный узел CRM)**

Здесь сходятся данные из сайта и бота.

* `id`: uuid (PK)  
* `expert_id`: uuid (FK)  
* `telegram_id`: text (Для связи с ботом)  
* `data`: jsonb (Ответы на квиз, результаты калькулятора)  
* `score`: int (Баллы горячести лида)  
* `status`: lead\_status (Enum: `new`, `qualified`, `paid`, `archive`)  
* `source_id`: uuid (FK к таблице источников)

#### **4\. `products` (Продуктовая лестница)**

* `id`: uuid (PK)  
* `expert_id`: uuid (FK)  
* `type`: product\_type (Enum: `tripwire`, `main`, `premium`)  
* `price`: numeric  
* `metadata`: jsonb (Описание, бонусы, ссылки на оплату)

---

## **3.2. Логика Edge Functions (The Logic Layer)**

Серверные функции Supabase выполняют «грязную работу» в фоновом режиме.

* **`on_new_lead`:** Срабатывает мгновенно при появлении записи в `leads`. Отправляет уведомление Сергею в Telegram: «Босс, новый горячий лид\! Скоринг: 85\. Ссылка на CRM...».  
* **`track_click`:** Обрабатывает короткие ссылки (osipuk.ru/go/...). Фиксирует переход, увеличивает счетчик в таблице `sources` и перенаправляет пользователя на целевой блок сайта.  
* **`ai_consultant`:** Мост между ботом и Vertex AI. Передает ответы пользователя из БД в ИИ, получает персональный «Код монетизации» и сохраняет его обратно в профиль лида.

---

## **3.3. Безопасность и RLS (The Fortress)**

Чтобы твои данные и данные твоих клиентов были в безопасности, мы внедряем Row Level Security:

* **Анонимный доступ:** Разрешен только `SELECT` для таблиц `m_content_blocks` и `products`, чтобы сайт мог загружаться у всех.  
* **Доступ лида:** Клиент может только делать `INSERT` в таблицу `leads` (оставлять заявку), но не может читать чужие данные.  
* **Доступ эксперта:** Только авторизованный Сергей (или другой эксперт) видит свои данные через фильтр по `expert_id`.

# **Часть 4: Интеллект Системы (Vertex AI Engine)**

Этот раздел описывает настройку нейросетевого слоя, алгоритмы обработки данных клиента и логику генерации персональных стратегий.

## **4.1. Конфигурация «Золотого промпта» (The DNA Prompt)**

Системная инструкция (System Instruction) определяет личность Агента. Она базируется на твоей философии «Монетизатора».

* **Роль и Тон (Persona):** Агент — это не «помощник», а **Монетизатор ресурсов**. Тон общения: прямой, экспертный, мужской, без «воды» и пустой мотивации.  
* **Фундаментальные правила (The Core Logic):**  
  * «Реклама не исправляет хаос — она его усиливает». Сначала ищем деньги в базе, потом в трафике.  
  * Любая стратегия должна опираться на **7 скрытых источников прибыли** (База, Повторные продажи, Рекомендации и др.).  
* **Словарный запас:** Обязательное использование терминов: «спящие активы», «деньги под ногами», «таблетка от боли», «паутина возможностей».

## **4.2. Механика RAG: Обучение на живом опыте**

Чтобы Агент не галлюцинировал, мы используем технологию **RAG (Retrieval-Augmented Generation)**.

1. **Библиотека кейсов:** Все твои 6 эталонных кейсов (Аня-фотограф, Юрист, BBQ и др.) хранятся в векторной базе данных.  
2. **Семантический поиск:** Когда клиент пишет «у меня юридическая фирма», ИИ не просто генерирует текст, а сначала находит кейс №5 и на его основе строит рекомендации.  
3. **Контекстная адаптация:** ИИ адаптирует твой прошлый опыт под текущую ситуацию клиента, сохраняя логику «входного продукта» и «окон в расписании».

## **4.3. Алгоритм генерации «Кода монетизации»**

После прохождения квиза на сайте или в боте, ИИ запускает цикл генерации персонального отчёта, состоящего из трёх блоков:

* **Блок 1: Диагностика активов.**  
  * Идентификация «Зоны недозаработка».  
  * Пример: «Твой главный актив — база старых клиентов (2000 чел.), которая сейчас является архивом, а не деньгами».  
* **Блок 2: Математика потерь (The Loss Matrix).**  
  * Расчёт по формуле «Цены бездействия» на основе введённых данных.  
  * Визуализация: «Из-за отсутствия системы повторных продаж ты теряешь \[X\] рублей ежемесячно».  
* **Блок 3: План активации на 7 дней.**  
  * Три конкретных шага по твоей методике.  
  * Пример: «Шаг 1 — Выдели VIP-клиентов. Шаг 2 — Сделай полезный "шаг навстречу" (подарок/сервис). Шаг 3 — Собери простую таблицу контроля».

## **4.4. Квалификационный фильтр (Lead Scoring Logic)**

ИИ-Агент не просто консультирует, он «просеивает» лидов для тебя. В процессе диалога он начисляет баллы в CRM:

* **Бизнес-зрелость:** Если у клиента есть продукт, база и продажи — **высокий балл**.  
* **Запрос на халяву:** Если клиент ищет «волшебную кнопку» или не готов брать ответственность — **минус баллы** (отсекаем в архив).  
* **Готовность к действию:** Если клиент подтверждает готовность внедрить «шаг навстречу» прямо сейчас — **приоритет в CRM**.

# **Часть 5: Frontend — Реализация 11 блоков (The Matrix Interface)**

Этот раздел описывает, как технически реализовать каждый из 11 блоков твоего контента, соблюдая протоколы дизайна и логику конверсии.

## **5.1. Общие принципы верстки**

* **Стек:** React \+ Tailwind CSS \+ Framer Motion (для плавных появлений и «скользкой дорожки»).  
* **Стиль:** Matrix Protocol (Obsidian Deep фон, Matrix Gold для цифр, Matrix Emerald для кнопок).  
* **Адаптив:** Строго Mobile-first (всё должно идеально нажиматься пальцем).

## **5.2. Детальная спецификация блоков**

### **Блок 1\. Главный экран \+ Умный Квиз**

* **Визуал:** Крупный заголовок Satoshi Black. Под ним — кнопка \[НАЙТИ МОИ ТОЧКИ РОСТА\] с пульсирующим изумрудным свечением.  
* **Логика Квиза:** При нажатии на кнопку открывается полноэкранный интерфейс квиза.  
  * **Микро-инсайты:** После выбора ответа в каждом вопросе (1–5) блок с вопросом плавно сдвигается вверх, и снизу появляется блок «Инсайт» на Glassmorphism-подложке. Кнопка «Далее» появляется только после прочтения инсайта.  
  * **Сбор данных:** Имя пользователя запрашивается на финальном шаге. Данные отправляются в Supabase (`leads`) методом `upsert` по мере прохождения, чтобы не потерять лид.

### **Блок 2\. Лидер среды (Об эксперте)**

* **Верстка:** Фото Сергея в стиле «бизнес-портрет» слева, текст в стиле Огилви справа.  
* **Акценты:** Дипломы и регалии оформлены в виде «бегущей строки» или компактных карточек с иконками Matrix Gold.

### **Блок 3\. Твердые результаты (Кейсы)**

* **Техзадание для разработки:** Сетка из 6 карточек (загружаются из `m_cases`).  
* **Интерактив:** При клике на превью карточка расширяется (Modal или Accordion), открывая «Подробную часть» кейса.  
* **Цифры:** Все показатели (+55 000 ₽, 10 обращений и т.д.) выделены шрифтом **Geist Mono** и золотым цветом.

### **Блок 4\. Философия Монетизатора (Манифест)**

* **Дизайн:** Темный блок с акцентным типографским оформлением. Текст «Реклама не исправляет хаос...» — самый крупный заголовок в блоке.  
* **Элемент:** Использование буллитов в виде перечеркнутых символов (против хаоса) и изумрудных стрелок (за порядок).

### **Блок 5\. Метод: 7 источников скрытой прибыли**

* **Интерактив:** «Рентгеновское зрение». 7 иконок/блоков. При наведении на источник (например, «Клиентская база») — он подсвечивается изумрудом, и появляется краткое описание механики.

### **Блок 6\. Анти-цель: Жесткий фильтр**

* **Стиль:** Лаконичный, «холодный» дизайн. Использование Obsidian Deep и Zinc/60.  
* **Цель:** Создать психологический барьер, чтобы кнопку «Я в адеквате» нажимали только целевые люди.

### **Блок 7–9. Эшелоны услуг**

* **Верстка:** Три вертикальных или горизонтальных карточки (Диагностика \-\> Среда \-\> Масштаб).  
* **Кнопки:** Каждая ведет на единую модальную форму «Оставить заявку».  
* **Статусы:** Для тарифа «Наставничество» (50к) добавить плашку «Мест мало» или «Только после интервью».

### **Блок 10\. Подарок (The Irresistible Offer)**

* **Дизайн:** Самый яркий блок. Золотой градиент на фоне или в обводке карточки.  
* **Счетчик:** Динамический текст: «Осталось 2 места на этой неделе» (редактируется из админки).

### **Блок 11\. Контакты**

* **Функционал:** Кнопки перехода в ТГ-личку и ТГ-канал. Ссылка на канал должна иметь UTM-метку для отслеживания переходов.

# **Часть 6: Interaction Layer & Bot Logic (Интеграция и Магия)**

Этот раздел описывает, как технически реализовать переход пользователя с сайта в бот и как ИИ-Агент должен обработать полученные данные для выдачи «Персональной карты активации».

## **6.1. Механика «Бесшовного перехода» (Deep Linking)**

Чтобы бот «узнал» клиента по имени и вспомнил его ответы на квиз, мы используем систему динамических ссылок.

* **Trigger:** На финальном экране квиза (после ввода имени) кнопка \[ПОЛУЧИТЬ РЕЗУЛЬТАТ В TELEGRAM\] ведет по ссылке: `https://t.me/Osipuk_Bot?start=UUID_сессии`.  
* **Logic:** 1\. При нажатии бот получает уникальный `UUID`. 2\. Бот мгновенно делает запрос в Supabase в таблицу `leads`, вытягивая имя и все 5 ответов на вопросы. 3\. **Первое сообщение бота:** «\[Имя\], приветствую\! Я уже изучил твой профиль. Твой главный "спящий" актив — \[Название ресурса из вопроса 2\]. Сейчас я сформирую твою персональную карту...».

## **6.2. Алгоритм генерации «Карты активации прибыли» (AI Synthesis)**

# **Часть 8: Маркетинг и Трафик-центр (The Growth Engine)**

Этот раздел описывает инструменты привлечения трафика, систему атрибуции (отслеживания источников) и аналитический хаб.

## **8.1. Система «Умных ссылок» (Short-Link Service)**

Вместо длинных и некрасивых UTM-ссылок, система генерирует эстетичные короткие адреса, которые не стыдно отправить в личку или разместить в сторис.

* **Формат:** `osipuk.ru/go/[alias]` (например, `osipuk.ru/go/forum_spb`).  
* **Логика редиректа:** 1\. Пользователь кликает по ссылке. 2\. **Edge Function** в Supabase перехватывает клик. 3\. Записывает в таблицу `traffic_logs`: время, IP, User-Agent и `source_id`. 4\. Перенаправляет пользователя на сайт, автоматически вшивая `source_id` в локальное хранилище браузера (LocalStorage).  
* **Связка с лидом:** Если после перехода по ссылке пользователь проходит квиз, в его карточку в CRM автоматически записывается: «Источник: Форум СПБ».

## **8.2. Генератор Брендированных QR-кодов**

Поскольку ты часто работаешь «в полях», каждый QR-код должен быть продолжением дизайна **The Money Matrix**.

* **Технические требования:**  
  * Формат: SVG (для печати любого размера).  
  * Цвета: Основной — `Obsidian Deep`, узоры — `Matrix Emerald` или `Matrix Gold` (выбирается в админке).  
  * Логотип: Возможность вставки логотипа «Монетизатор» в центр кода.  
* **Сценарии использования:** Генерация кода прямо в админке для визиток, презентаций или тейбл-тентов на мероприятиях.

## **8.3. Партнёрский и Реферальный модуль**

Твой нетворкинг — это твоя армия продавцов.

* **Персональные кабинеты партнёров:** В админке ты создаёшь запись для партнёра (например, «Алексей»).  
* **Трекинг выплат:** Система считает, сколько лидов привел Алексей и сколько из них купили «Блиц» или «Диагностику».  
* **Авто-отчёты:** Партнёр может получить свою статистику через специальный мини-интерфейс или через бота, видя, как его рекомендации превращаются в результат.

## **8.4. Аналитический хаб: Метрики эффективности**

Интерфейс, который заменяет тебе десятки таблиц.

* **Метрика 1: ROI Рукопожатия.** Сколько прибыли принес конкретный канал (например, «Нетворкинг в Клубе Х») относительно потраченного там времени или денег.  
* **Метрика 2: Воронка Квиза.** Поэтапный отвал на каждом из 5 вопросов. Если на 3-м вопросе уходит 40% — мы это видим и меняем текст вопроса в админке.  
* **Метрика 3: Heatmap «Золотых кнопок».** Статистика нажатий на кнопки разных эшелонов услуг. Ты видишь, какой оффер «цепляет» лучше.

## **8.5. Сквозная аналитика: Сайт \-\> Бот \-\> Продажа**

Система объединяет данные из двух миров:

1. **Web-аналитика:** Кто зашёл, откуда, что нажал.  
2. **Bot-аналитика:** Кто подписался, кто скачал «Карту активации», кто застрял на этапе записи.

**Результат для тебя:** Ты открываешь админку и видишь: «Пост в канале А принес 100 переходов, 20 лидов, 5 продаж Блица на сумму 9950 ₽. Стоимость лида — 0 ₽».
